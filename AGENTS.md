# AGENTS.md

## Task Context
We are writing a paper about the economics of spec-driven software engineering and a mental model for organizing SDD work in a large, mature organization.

## Audience
Write for project managers, engineering managers, and tech leads.
Assume the readers already understand agile delivery frameworks such as Scrum, Kanban, and SAFe, but may not have a shared vocabulary for spec-driven development.

## Goal
Create a common mental model that helps the audience:
- Compare experiences and opinions using the same terms.
- Reason about organization design for SDD.
- Discuss human work versus agent work.
- Discuss labor allocation versus token allocation.
- Identify where SDD improves flow, quality, and coordination, and where it introduces new tradeoffs.

## Writing Guidance
- Be clear, analytical, and practical.
- Prefer concepts, definitions, and decision frameworks over slogans.
- Use consistent terminology throughout the paper.
- Distinguish between observation, interpretation, and recommendation.
- Treat SDD as an organizational and operational model, not only a coding technique.
- Keep the tone suitable for a mature enterprise audience.

## Format
- Use Markdown format for the paper with Mermaid for diagrams.
- Use headings and subheadings to structure the paper.
- Include diagrams or tables if they help clarify concepts.
- Use bullet points for lists of related ideas or tradeoffs.
- Provide examples or case studies to illustrate key points.

## Framing Constraints
- Do not assume greenfield teams or startup conditions.
- Account for large-team coordination, governance, and dependency management.
- Fit the discussion into existing agile operating models rather than replacing them by default.
- Be explicit about tradeoffs between speed, control, quality, predictability, and autonomy.

## Content Priorities
- Define what spec-driven software engineering means in this context.
- Explain the mental model in a way that supports debate and shared understanding.
- Make room for the economics of human effort, AI/agent effort, and token consumption.
- Highlight organizational implications for planning, review, verification, and accountability.
- Consider the `notes.md` file as a source of raw material and research ideas.

## Working Rule
Before drafting major sections, check that each section advances the shared mental model rather than simply restating implementation details.
